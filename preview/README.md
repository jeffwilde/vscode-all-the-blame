# Preview

A static, browser-only demo of the libgit2-via-WASM blame stack.

## What it shows

Loads a small fixture git repo (4 commits across 4 authors, all on `demo.ts`)
into the wasm-git module's in-memory filesystem, then runs `lg2_blame_stream`
and renders results live as the algorithm walks history backward.

Everything happens in the browser tab. There is no backend, no `git` binary,
no GitHub login, no Codespace. Works in any modern browser.

## Run locally

```bash
pnpm run preview
```

Then open <http://localhost:8080>.

## Hosted

Deployed to GitHub Pages from `main` via `.github/workflows/preview.yml`.

URL: <https://jeffwilde.github.io/vscode-all-the-blame/>

## What's in this directory

- `index.html` — page skeleton
- `style.css` — styles
- `app.js` — the bootstrap that loads the WASM module, seeds MEMFS,
  runs streaming blame, and updates the DOM as events arrive
- `lg2.js` / `lg2.wasm` — the forked wasm-git build with our blame
  exports. Copied from `vendor/wasm-git/` by `pnpm build-preview`.
- `fixture.js` — the demo git repo packed as a base64-encoded tar.
  Generated by `scripts/build-preview-fixture.mjs`. Re-run via
  `pnpm build-preview-fixture`.

## Caveats

This preview demonstrates the WASM blame stack. It does **not** load the
full VS Code extension (status bar, decorations, settings UI). That
integration is wired in `src/web-entry.ts` and ships in the extension
VSIX that runs inside `vscode-web`; running that here too would require
embedding `vscode-web` itself, which is a much heavier static deployment
(tens of MB) and out of scope for this preview.

If you want to test the extension inside `vscode-web`, install the VSIX
into `vscode.dev` once it's published.
